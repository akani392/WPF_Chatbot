The Cybersecurity Awareness Assistant is an interactive desktop application developed using Windows Presentation Foundation (WPF) and C# (.NET). It is designed as an educational tool to instruct users on five critical domains of information security: phishing, malware, password vulnerabilities, firewalls, and digital scams. Rather than managing complex window navigation or dynamic page frames, the user interface relies on a single-window state view pattern. The main dashboard wraps all visual controls inside a uniform, full-screen parent grid containing three independent layer grids that act as individual application states. The application completely controls transitions through backend property manipulation of the visibility enum, shifting between the welcome screen (logo_grid), the onboarding screen (username_grid), and the central chat interface (chats_grid). To maintain responsive layout mechanics and prevent structural breaking when the window is resized, elements avoid fixed pixel margins; instead, they utilize vertical stack panels for standard cascading and explicit grid row and column definitions so the chat history feed expands elastically while the input controls remain anchored to the base framework.
The backend code-behind functions as a deterministic text-processing pipeline that coordinates data using strongly-typed generic collections. Textual inputs are standardized to lower-case using the string formatting method .ToLower() to prevent matching blindspots before being processed by a reverse lookup dictionary (topicKeyWords) for exact keyword substring matching. Once a core topic is identified, a multidimensional string array repository (chatbotResponses) supplies the educational content, selecting a unique statement via a pseudo-random seed generator (Random.Next()) to simulate a natural conversational exchange.

Dialogue context and system state tracking are split between volatile session memory and persistent file systems. Volatile tracking uses a counter dictionary (topicAskCount) to monitor how frequently a topic has been discussed; when a single threshold reaches exactly three queries, the engine intercepts the normal message flow, instantiates an intercept dictionary (activeMenu), and forces an interactive menu onto the user interface stack that maps direct numerical choices ("1", "2", or "3") to deep-dive payloads (subTopicResponses). Persistent storage handles explicit user declarations, such as expressions containing the phrase "interested in," by passing the string to a custom file handler (SaveToFile()) that strips out conversational variations, isolates the targeted topic string, and stream-writes it directly to an external storage file (memory.txt). At runtime, the system uses native file checking wrappers (File.Exists()) to verify this database and restore user personalization features. Furthermore, upon successful user onboarding, the application initiates a multimedia verification layer by invoking low-level Windows media pipelines (System.Media.SoundPlayer) to locate, load, and play a localized, uncompressed audio waveform configuration file (.wav).

Defensive programming constructs protect the core operational stability of the system against unhandled exceptions and invalid input strings. User inputs undergo immediate whitespace trailing elimination via .Trim() to remove empty leading and trailing characters prior to logic execution. Structural identity verification handles name onboarding by executing safe evaluations with string.IsNullOrWhiteSpace() and passing the string through an atomic regular expression compiled validator (Regex.IsMatch(username, @"^[a-zA-Z]+$")).
